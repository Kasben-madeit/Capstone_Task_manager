# Task Manager API

This repository contains a simple back‑end API for managing personal tasks. It
is built with **Django** and **Django REST Framework** and provides
functionality to register users, authenticate them, and allow them to create,
read, update and delete tasks. Tasks can optionally be organized into
categories.

## Features

- User registration and token‑based authentication (see the notes in
  [`settings.py`](task_manager/task_manager/settings.py)).
- CRUD operations for **categories** and **tasks** via REST endpoints.
- Each task belongs to a user (the `owner`); users can only interact with
  their own tasks.
- Categories are global and may be shared among users, making it easy to
  categorize tasks by common themes.
- A simple SQLite database for easy local development.

## API Endpoints

The API is exposed under the `/api/` prefix. Example routes:

| Method | Endpoint                | Description                                      |
|-------|-------------------------|--------------------------------------------------|
| POST   | `/api/categories/`      | Create a new category                             |
| GET    | `/api/categories/`      | List all categories                              |
| GET    | `/api/categories/{id}/` | Retrieve a specific category                     |
| PUT    | `/api/categories/{id}/` | Update a category                                |
| DELETE | `/api/categories/{id}/` | Delete a category                                |
| POST   | `/api/tasks/`           | Create a new task (requires authentication)      |
| GET    | `/api/tasks/`           | List all tasks belonging to the authenticated user |
| GET    | `/api/tasks/{id}/`      | Retrieve a specific task                         |
| PUT    | `/api/tasks/{id}/`      | Update a task                                    |
| DELETE | `/api/tasks/{id}/`      | Delete a task                                    |

## Running Locally

> **Note:** The environment provided here for demonstration does not ship
> Django or Django REST Framework out of the box. You will need to install
> these dependencies before running the project locally. If you only want to
> inspect the code structure, you can skip these steps.

1. Create and activate a virtual environment:

   ```bash
   python3 -m venv .venv
   source .venv/bin/activate
   ```

2. Install dependencies:

   ```bash
   pip install django djangorestframework
   ```

3. Run database migrations:

   ```bash
   python manage.py migrate
   ```

4. Create a superuser (optional for admin access):

   ```bash
   python manage.py createsuperuser
   ```

5. Start the development server:

   ```bash
   python manage.py runserver
   ```

The API will be accessible at `http://127.0.0.1:8000/api/`.

## Authentication

This project is configured to use Django's session authentication by default.
For token‑based authentication (suitable for APIs), you can install and
configure `rest_framework.authtoken` and obtain tokens via the `/api-token-auth/`
endpoint. Refer to the Django REST Framework documentation for details.
